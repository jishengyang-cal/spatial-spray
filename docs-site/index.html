Cross-device AR creation and discovery scaffold

Spatial Spray

Location-aware AR spray creation workflow for iPhone and Apple Vision Pro.

Spatial Spray 是一个跨设备 AR 喷漆创作与发现产品脚手架。它把社交登录、唯一用户名、附近作品发现、 AR 喷漆笔刷模型、UGC moderation 和 Apple 原生构建请求拆成可验证的 contracts、API、web simulator 和 Mac Builder workflow。当前目标是验证产品闭环和工程边界，不声称已经是生产级社交网络或 App Store ready 应用。

Identity before publishing

创作前先建立用户身份和唯一用户名。

Apple、Google、Facebook 登录在 dev-mode adapter 中建模，生产 provider token 验证仍属于后续配置和服务端工作。

Location-indexed AR pieces

喷漆作品以位置和 surface anchor 为核心对象。

API 和 contracts 记录 geo point、surface pose、spray strokes、visibility 和 moderation state。

Moderated UGC workflow

从第一版就把举报、屏蔽、删除和位置 denylist 纳入模型。

AR UGC 不是单纯画笔效果，必须同时处理附近发现、公共空间、安全和审查流程。

Problem

AR 社交创作不是只有渲染效果，还要处理身份、位置和公共空间治理。

用户在真实室内、室外和公共建筑附近创作 AR 内容时，产品必须回答谁发布了内容、作品在哪里、其他用户如何发现、 不当内容如何举报、某些地点是否禁止创作，以及 iPhone 和 Vision Pro 如何看到同一套数据。

Native iOS 和 visionOS 构建仍然依赖 macOS、Xcode、ARKit、RealityKit、签名资产和设备测试。这个 repo 先用 Linux-runnable API、contracts 和 web simulator 验证产品规则，再通过 Mac Builder workflow 交给 Apple 工具链。

Product risk it reduces

在投入真实设备、定位精度、App Store review 和生产登录 provider 前，先验证身份、附近发现、喷漆模型和 moderation workflow 是否可成立。

What it is

一个跨设备 AR UGC 产品的工程骨架。

Spatial Spray 把核心产品循环拆成 API、brush engine、web simulator、native Apple scaffold 和 Mac Builder handoff。 这让产品规则可以在 Linux 中验证，原生 AR 能力再由 Apple 平台承担。

Identity and username

账户和身份边界

Dev-mode provider login、refresh session 和 username reservation 支撑登录后创建唯一用户名的产品流程。

Spray domain model

喷漆作品模型

Spray piece 包含 owner、geo point、anchor payload、surface pose、strokes、visibility 和 moderation status。

Brush engine

真实喷漆效果原型

Brush engine 建模 particles、overspray、opacity、nozzle profile、drips 和 AR decal mesh data。

Moderation workflow

UGC 治理入口

Report、block、owner delete、admin hide/remove、audit log 和 location denylist 从 scaffold 阶段进入系统。

What it is not

当前仓库是产品和工程闭环 scaffold，不是生产承诺。

边界写清楚可以避免把 web simulator、dev login 和 native scaffold 误解成上线能力。

不是生产社交网络后端；当前 API 使用本地持久化状态。
不是 App Store ready 应用；仍需要真实签名、隐私材料、设备测试和 review 准备。
不是真实 provider 登录集成；生产必须服务端验证 Apple、Google 和 Facebook token。
不是全球厘米级 AR 锚定系统；MVP 以 manual-local anchors 和 surface pose 建模。
不是浏览器中的 ARKit 替代品；web simulator 用于产品和 API 验证。
不鼓励现实世界涂鸦或破坏财产；作品是数字 AR 内容。

Architecture

Current implementation

当前主干中的可验证落点

这些能力都能在 repo 的 contracts、API、brush engine、web simulator、native scaffold、tests 或 docs 中找到支撑。

Auth and username

Dev provider login adapters model Apple, Google and Facebook sign-in flow with refresh sessions and unique username reservation.

Spray piece API

Persistent local API supports spray creation, nearby lookup, clusters, visibility, owner delete and location denylist state.

Brush engine

Spray stroke model covers colors, nozzle, particles, overspray, opacity, drips, material and AR decal mesh data.

Moderation primitives

Report, block user, admin hide/remove and audit log flows are available in API and simulator paths.

Web simulator

Simulator validates login, username, map discovery, clusters, camera spray canvas and moderation actions.

Native Apple scaffold

Native source scaffold covers iPhone AR spray and Vision Pro mixed immersive viewing boundaries.

Mac Builder workflow

Mock builder and client scripts validate iOS/visionOS build request submission without requiring Xcode on Linux.

Compliance docs

Privacy, content policy, compliance and App Store review notes document product responsibilities before release work.

Operator workflow

本地产品闭环和 Mac Builder 验证流程

本地流程验证 contracts、API、web simulator 和 E2E；native build request 先走 mock builder，真实 iOS/visionOS build 仍需要远程 Mac。

  1. 安装依赖：`pnpm install`。
  2. 运行 workflow 检查：`pnpm workflow:check`。
  3. 运行测试：`pnpm test`。
  4. 启动本地 API 和 web simulator：`pnpm dev`。
  5. 启动 mock builder：`pnpm native:mac-builder:mock`。
  6. 提交 visionOS build request：`SPATIAL_SPRAY_MAC_BUILDER_URL=http://127.0.0.1:4391 pnpm native:visionos-build:submit`。
